controllers/get-users.ts 
import { createHandler } from '@chaeco/auto-router'

export default createHandler(
  async (ctx) => {
    const users = await db.users.findMany()
    ctx.res.body = { users }
  },
  { description: 'List all users', tags: ['Users'] }
)

// → GET /api/users

Everything you need

Powerful features that make routing effortless. No magic, just sensible defaults.

Zero Configuration

Just point to your controllers directory. Routes are automatically discovered from file structure.

Nested Directories

Build complex route hierarchies with nested folders. Path parameters work in both files and directories.

Auth Metadata

Built-in permission system with forcePublic and forceProtected patterns for fine-grained control.

TypeScript First

Full generic support with RouteHandler<TCtx, TRes>. Works with any framework's context type.

Multi-Prefix Support

Register the same controllers under multiple prefixes for API versioning or locale-based routing.

Edge Runtime Ready

staticAutoRouter for Cloudflare Workers, Deno Deploy, and bundled apps without filesystem access.

Quick Start

Get started in minutes. Just two steps.

1

Create your controllers

File names define HTTP routes. Use [param] for dynamic segments.

📁 controllers/
📄 get-users.ts → GET /api/users
📄 get-[id].ts → GET /api/:id
📄 post-login.ts → POST /api/login
📁 users/[userId]/posts/
📄 get.ts → GET /api/users/:userId/posts
2

Mount the router

Just one line to register all routes automatically.

app.ts 
import { autoRouter } from '@chaeco/auto-router'

const app = createApp()

app.extend(
  autoRouter({
    dir: './controllers',
    prefix: '/api',
  })
)

app.listen(3000)

API Reference

Clean, minimal API. Three main exports.

autoRouter(options) Core

Factory function that returns a plugin for app.extend(). Scans your controller directory and registers all routes automatically.

Options

dir Controller directory path
prefix Route prefix (string or array)
defaultRequiresAuth Global auth default
forcePublic Always-public route patterns
forceProtected Always-protected route patterns
onLog Custom log callback
createHandler(handler, meta?) Helper

Wraps a handler with metadata. Use when you need per-route auth or documentation info.

Meta Fields

requiresAuth Boolean auth override
description Human-readable description
tags For API documentation grouping
[key: string] Any custom metadata
app.$routes Registry

Access all registered routes after loading. Perfect for auth middleware and API doc generation. 

// After app.extend(autoRouter(...))
app.$routes.all             // RouteInfo[] — every route
app.$routes.publicRoutes    // { method, path }[]
app.$routes.protectedRoutes // { method, path }[]
